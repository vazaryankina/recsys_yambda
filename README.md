# Sequential Music Recommendation and Feedback Loop Analysis

Проект посвящён построению и сравнению рекомендательных моделей для задачи **next-track prediction** на музыкальных пользовательских последовательностях. Основной фокус работы — сравнить классические baseline-модели и нейросетевые sequential-модели, а затем использовать лучшую SASRec-модель для анализа различий между organic и non-organic взаимодействиями.

## Цель проекта

Цель исследования — построить последовательную рекомендательную модель для предсказания следующего трека пользователя и проанализировать, как разные типы пользовательских взаимодействий могут влиять на качество и свойства рекомендаций.

В работе рассматриваются:

- формирование репрезентативного сабсэмпла из большого event-level датасета;
- построение leave-one-out split для задачи next-item prediction;
- сравнение простых baseline-моделей, GRU и SASRec;
- fine-tuning SASRec как основной модели;
- анализ feedback loop через сравнение моделей, обученных на organic и non-organic interactions;
- оценка моделей не только по accuracy-метрикам, но и по coverage, diversity и popularity bias.

## Структура репозитория

Репозиторий состоит из пяти основных ноутбуков.

### 1. `notebook_1_final_subsample_selection.ipynb`

В первом ноутбуке формируется финальный reduced dataset для дальнейших экспериментов.

Основные шаги:

- анализ полной совокупности listening-событий;
- построение поведенческих признаков пользователей;
- стратифицированный отбор пользователей;
- отбор треков с сохранением распределения по popularity bins;
- проверка репрезентативности по пользователям, трекам, organic / non-organic share и played ratio;
- сохранение финального `final_events.parquet`.

Итог: сформирован сабсэмпл примерно из 5000 пользователей, 300 000 треков и 10.2 млн событий.

### 2. `notebook_2_eda_baselines_gru.ipynb`

Во втором ноутбуке проводится compact EDA финального сабсэмпла, строится leave-one-out split и обучаются baseline-модели.

Рассматриваются модели:

- Most Popular;
- Last Item;
- Item-to-Item;
- GRU baseline.

Метрики:

- Recall@K;
- Precision@K;
- NDCG@K;
- MRR@K;
- Catalog Coverage@K;
- Gini@K;
- Top Popularity Bin Share@K.

Итог: среди простых baseline-моделей лучшим оказался Item-to-Item, что показывает важность локальных переходов между треками в задаче next-track prediction.

### 3. `notebook_3_sasrec_experiments.ipynb`

В третьем ноутбуке реализуется и тестируется SASRec как основная sequential-модель.

Основные шаги:

- построение sequence dataset;
- реализация SASRec на PyTorch;
- обучение базовой SASRec-модели;
- проверка усиленной архитектуры;
- тестирование варианта с negative sampling;
- сравнение SASRec-кандидатов по ranking-метрикам и метрикам покрытия.

Итог: SASRec показывает перспективное качество, но требует дополнительного fine-tuning. Лучшие кандидаты переходят в следующий ноутбук.

### 4. `notebook_4_sasrec_final_finetuning.ipynb`

В четвёртом ноутбуке проводится финальный fine-tuning SASRec.

Проверяются несколько направлений улучшения:

- увеличение числа обучающих состояний на пользователя;
- более длительное обучение;
- увеличение длины пользовательской истории;
- более сильная attention-архитектура;
- более широкое hidden representation.

Финальная модель выбирается по validation Recall@50, после чего оценивается на test.

Итог: фиксируется финальная SASRec-модель, которая используется в дальнейшем feedback loop analysis.

### 5. `notebook_5_feedback_loop_analysis.ipynb`

В пятом ноутбуке анализируется feedback loop через сравнение моделей, обученных на разных типах взаимодействий.

Сравниваются две SASRec-модели:

- SASRec, обученная только на organic interactions;
- SASRec, обученная только на non-organic interactions.

Анализируются:

- качество предсказания;
- различия по organic и non-organic test targets;
- coverage;
- diversity;
- concentration / Gini;
- popularity bias;
- распределение рекомендаций по popularity bins.

Итог: модели, обученные на organic и non-organic interactions, дают разные свойства рекомендаций. Это позволяет обсуждать возможный feedback loop и концентрацию рекомендаций вокруг популярных треков.

## Используемые метрики

В проекте используются две группы метрик.

### Ranking / accuracy metrics

Эти метрики показывают, насколько хорошо модель предсказывает следующий трек пользователя:

- `Recall@K` — попал ли правильный трек в top-K рекомендаций;
- `Precision@K` — доля релевантных объектов в top-K;
- `NDCG@K` — учитывает позицию правильного трека в рекомендательном списке;
- `MRR@K` — обратный ранг первого правильного попадания.

Основной горизонт сравнения — `@50`, при этом `@10` также сохраняется как дополнительная проверка.

### Exposure / diversity / popularity metrics

Эти метрики описывают свойства самих рекомендаций:

- `Catalog Coverage@K` — какую долю каталога модель покрывает рекомендациями;
- `Unique Recommended Items@K` — сколько уникальных треков появляется в рекомендациях;
- `Gini@K` — насколько рекомендации сконцентрированы на малом числе объектов;
- `Top Popularity Bin Share@K` — какая доля рекомендаций приходится на самые популярные треки.

Эти метрики важны, потому что модель может иметь хорошее качество по Recall, но при этом рекомендовать слишком узкий и слишком популярный набор треков.

## Методология

В работе используется постановка **next-item prediction**. Для каждого пользователя строится последовательность прослушиваний, а задача модели — предсказать следующий трек.

Для оценки используется **leave-one-out split**:

- последнее событие пользователя — test target;
- предпоследнее событие — validation target;
- все более ранние события — train history.

Такой подход позволяет оценивать модели в последовательной постановке и сравнивать все модели в одинаковых условиях.

## Основные выводы

1. Item-to-Item является сильным baseline для музыкальных последовательностей, так как хорошо улавливает локальные переходы между треками.
2. GRU извлекает сигнал из пользовательской истории, но в простой конфигурации уступает Item-to-Item.
3. SASRec позволяет работать с более длинным пользовательским контекстом и является основной моделью для дальнейшего анализа.
4. Метрики coverage, Gini и Top Popularity Bin Share показывают, что важно оценивать не только точность рекомендаций, но и их разнообразие.
5. Organic и non-organic взаимодействия могут приводить к разным свойствам рекомендательных списков, что важно для анализа feedback loop.
